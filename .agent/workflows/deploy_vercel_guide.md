# クライアント納品（Vercel公開）サポート・ワークフロー

このワークフローは、開発者がDiscord等で通話しながらVTuber様（クライアント）へ納品・デプロイ手順を案内する際の「完璧なカンペ・ナビゲーション」として機能します。

## 事前準備（VTuber様へお願いすること）
通話の前に、以下の2つのアカウントを作成してログイン状態にしておいていただきます。
1. **GitHubアカウント**（コードを置く場所）
2. **Vercelアカウント**（GitHub連携で自動ログイン。サイトを公開する場所）

## 案内ステップ（通話時のスクリプト）

### Step 1: リポジトリの作成と連携（GitHub）
1. 「まずは、事前にお渡ししている『私たちのGitHubリポジトリのURL』を開いてください。」
2. 「画面右上に『Use this template』という緑色のボタン（または Fork ボタン）がありますので、そこをクリックしてください。」
3. 「『Create a new repository』を選び、Repository name に好きな名前（例：`my-vtuber-site`）を入力し、Create ボタンを押してください。」
4. 「これで、あなたのGitHubの中に、あなた専用のサイトのコードが入りました！」

### Step 2: YouTube / Twitch APIの取得
（※クライアントに取得してもらう場合。事前に開発者が発行して環境変数（.env）に渡す場合はスキップ）
1. 「次に、配信予定を自動で持ってくるための『鍵（APIキー）』を取りに行きます。こちらのURL（Google Cloud Console等）を開いてください…」
*(この部分はAPI取得のマニュアルページURLURLを案内します)*

### Step 3: Vercelへのデプロイ（公開）
1. 「最後にサイトを世界に公開します！ Vercel のダッシュボードを開いてください。」
2. 「画面右上の『Add New…』から『Project』を選んでください。」
3. 「リストの中に先ほどGitHubで作った `my-vtuber-site` があるので、『Import』を押します。」
4. 「『Environment Variables』（環境変数・秘密の言葉）の欄を開き、以下の内容を入力してください。
   - Name: `NEXT_PUBLIC_YOUTUBE_API_KEY` / Value: （Step2で取得した鍵）
   - Name: `NEXT_PUBLIC_VTUBER_CHANNEL_ID` / Value: （YouTubeチャンネルURLの末尾のID）」
5. 「入力が終わったら、青い『Deploy』ボタンを押して完了です！数十秒待つと、お披露目の花吹雪が舞います！」

### Step 4: 今後の運用について
1. 「このサイトは、さっき入れた鍵の力で、あなたの配信情報に自動で書き換わり続けます。」
2. 「ブログを書くような面倒な作業は一切ありません！活動に専念してください！」
