# クライアントサイト構築ワークフロー

マスターリポジトリの雛形を用いて、VTuber個人のアイデンティティ（色、設定、APIなど）を反映させた新しいサイト開発を開始します。

1. ターゲットVTuberの要件ヒアリング
ユーザーから、「誰向けか」「希望するデザインジャンル（和風・サイバー等）」「YouTube/Twitch ID」などの基本情報を聞き取ります。

2. テーマ（着せ替え）の一括適用
AIスキル `generate_theme` を実行し、ヒアリングしたジャンルに最適な配色を `userConfig.ts` と `tailwind.config.ts` (またはCSS変数) に反映させます。

3. API設定の適用
`src/config/userConfig.ts` に、対象VTuberのYouTube ID（およびTwitch ID）を設定します。

4. モックデータの作成（任意）
開発中の一時的なAPI制限を避けるため、必要に応じて AIスキル `mock_streaming_data` を実行し、ダミーデータでUIの表示調整を行えるようにします。

5. カスタムコンポーネントの調整
必要に応じて、ロード画面のスプラッシュアニメーションやメインビジュアル、立ち絵などのアセットを対象ユーザー向けに差し替えます。
