# アップストリーム同期ワークフロー

マスターリポジトリ（テンプレート大元）で行われた機能追加やバグ修正を、個々のクライアント（VTuber）環境へ安全にマージ・同期します。

// turbo
1. リモート（マスター）のフェッチ
マスターリポジトリの最新情報を取得します（既に `upstream` として追加されている前提）。
```bash
git fetch upstream
```

2. 変更の確認
マスター側の `main` または `dev` ブランチの変更内容をチェックします。特に、`src/core/` 以外の着せ替えファイル（`userConfig.ts`など）がコンフリクトしないか確認します。
```bash
git diff HEAD..upstream/main
```

// turbo
3. 変更のマージ
マスターの変更を現在のクライアント環境（現在のブランチ）にマージします。
```bash
git merge upstream/main
```

4. 依存関係の更新
パッケージが追加されている可能性があるため、クリーンインストールを行います。
```bash
pnpm install
```

5. コンフリクト解決と動作確認
もし `userConfig.ts` やカスタムUI部分でコンフリクトが発生した場合は手動で解決し、開発サーバーを起動して動作確認を行います。
