# generate_theme スキル実行手順

ユーザーから「テイストを変更して」「〇〇風にして」という指示（あるいはこのスキル自体の呼び出し）があった場合、以下の手順に沿って確実にプロジェクトのテーマ「着せ替え」を実行します。

## 1. ターゲットファイルの特定
以下の2つの主要ファイルが存在するか確認し、修正の準備をします。
- `src/config/userConfig.ts` : ユーザー固有の設定ファイル
- `tailwind.config.ts` (または `src/app/globals.css` 等): CSS変数が定義されているファイル

## 2. 配色の選定と思考プロセスの提示
ユーザーの要望（例：「和モダン」）に対して、最適な以下の色を選定し、まずは提案（Hex値）理由を明記して出力します。
- **Primary (メインカラー)**: サイトの印象を決定づける色
- **Secondary (サブカラー)**: Primaryを引き立てる色
- **Accent (アクセント)**: ボタンや強調したいテキストに使う色
- **Background/Foreground**: 背景色と基本テキスト色

## 3. ファイルの更新
選定した配色を元に、プロジェクトのルール（Rule.md）に従って以下の通りファイルを更新します。
- **`userConfig.ts` の更新**:
  `theme` や `colors` に関連する設定値があれば上書きします。
- **CSS変数の更新**:
  shadcn/ui は通常 `:root` にて変数を管理しています。提案した色をHSL値に変換し、プロジェクトのグローバルCSSの `:root` （および `.dark` があればダークモード）の変数を書き換えます。

## 4. 実行後の報告
変更したファイル名と、それぞれの変更内容（どんな色を設定したか）を簡潔にユーザーへ報告し、「どう変更されたか表示確認をお願いします」と伝えます。
