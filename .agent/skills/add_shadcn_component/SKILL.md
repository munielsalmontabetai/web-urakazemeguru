# add_shadcn_component スキル実行手順

1. **コンポーネントの追加コマンド実行**
   ターミナルツールを使用し、指定されたコンポーネントを追加します。
   実行するコマンド: `pnpm dlx shadcn-ui@latest add [コンポーネント名]`
   ※プロジェクトのルールにより、パッケージマネージャーは必ず `pnpm` を使用してください。

2. **追加されたコードの確認 (.tsx の読み込み)**
   `src/components/ui/[コンポーネント名].tsx` が作成されたことを確認し、ファイルを読み込みます。

3. **ルールに沿った魔改造（Tailwind調整）**
   Rule.mdの「着せ替え」要件を満たすため、追加されたコンポーネントのクラス名を以下の観点でチェック・修正します。
   - `bg-primary`, `text-primary-foreground` などのCSS変数を活用したクラスが正しく当たっているか。
   - レスポンシブ対応（`sm:`, `md:` など）が不足していないか（モバイルファーストの徹底）。
   - 固定の色（例: `bg-blue-500` のようなハードコードされた色）が使われていないか。使われている場合は、テーマ変数に置き換えます。

4. **クラス順序の整理（Prettier連携確認）**
   必要であれば、Tailwindのユーティリティクラス順序が公式推奨順になっているか確認し、手動で乱れがある場合は修正します（環境に `prettier-plugin-tailwindcss` が導入されていれば保存時に自動処理されます）。

5. **完了報告**
   追加されたコンポーネント名と、必要に応じてテーマに合わせたカスタム（魔改造）内容をユーザーに報告します。
