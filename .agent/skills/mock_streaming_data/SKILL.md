# mock_streaming_data スキル実行手順

1. **通信モジュールの特定**
   YouTube、またはTwitch APIと通信を行っているファイル（例: `src/core/youtube.ts` や `src/core/twitch.ts` など）を特定し、内容を読み込みます。

2. **Zodスキーマの確認**
   APIからのレスポンスを検証するために定義されている Zod のスキーマ定義（`LiveStreamSchema`, `VideoSchema` 等）を確認します。

3. **ダミーデータ JSON の生成**
   Zodスキーマに**完全に準拠**した、モック用のJSONデータ（あるいはTypeScriptのオブジェクト）を作成します。
   - 配信中（`live`）、配信予定（`upcoming`）、アーカイブ（`none`）の3つのステータスを網羅したデータを必ず用意してください（Twitchの場合の独自の配信ステータスやスケジュールデータ等も適宜含めます）。
   - タイトル等にはプレースホルダーを使わず、VTuberらしい具体的な文字列（例：「【初配信】はじめまして、〇〇です！」など）を使います。

4. **フェッチロジックの書き換え（切り替え機構の実装）**
   実際のフェッチロジックを一時的にコメントアウトするか、環境変数（例: `process.env.USE_MOCK_DATA === 'true'`）等によって実際のAPI通信とモックデータを容易に切り替えられるように書き換えます。

5. **完了報告**
   モックデータを作成し、API通信をモックに差し替えたことをユーザーに報告し、デザイン調整がクォータやリクエスト制限を気にせず行える状態になったことを伝えます。
